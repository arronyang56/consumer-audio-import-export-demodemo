# 用户视角审查记录

分支：`ux-user-perspective-review`

## 目标用户

不是所有同事都懂关务、税号、DG 或船期。这个网页应该像一个“进出口信息整合台”，让用户输入产品、国家、港口、船名或箱号后，先看到可执行结论，再决定是否打开原始网站复核。

## 当前主要问题

1. 首页和侧边栏模块太多，像系统设计图，不像查询工具。
2. 一些卡片展示了后台逻辑，例如数据源组合、固定清单、模拟说明，普通用户不需要先看到这些。
3. 政策和趋势结论偏模板化，输入不同国家/产品时差异不够明显。
4. 政策来源对小众国家覆盖不够，需要用全球通用数据源兜底，例如 WTO、World Bank、UN Comtrade、ITC、Google News、GDELT。
5. 船舶地图目前像示意图，用户期待真实地图底图和船舶坐标标记。
6. 通关状态入口和查询历史模块当前价值不高，应先移除；上海港放行查询后续做成独立脚本/半自动查询。
7. 港口风险需要更关注“港口 + 塞港/拥堵/等待靠泊/罢工/天气”等相关词，避免出现不相干新闻。
8. 产品范围不能只写 Consumer Audio；音频只是样例，查询能力要面向更广泛进出口货物。

## 本分支执行方向

1. 页面减法：移除模块总览、通关状态入口、查询历史、未来接口展示等偏内部逻辑模块。
2. 结论优先：政策、趋势、港口、进口国要求都先显示业务结论，不显示推理过程。
3. 数据源扩容：增加全球通用官方/媒体/航运来源，未知国家也能返回可用核验路径。
4. 船舶地图真实化：用公开地图瓦片显示船舶坐标，不再只展示模拟背景。
5. 邮件提醒低成本化：后端支持 SMTP 配置，后续可用 163 邮箱授权码测试。
